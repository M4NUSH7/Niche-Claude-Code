# Context bootstrap - before any scaffolding

**The harness has a hard prerequisite nobody wrote down: `context/` must exist and be true
before a single decision can be made.**

`harness.config.json` is derived from what the project *is* - domains, gates, sensitive paths,
doc routing. If the source material is a pile of loose files in a folder root, every one of those
derivations is a guess. So: **triage first, scaffold second.** Never invert this.

## Step 0 - classify the repo

```bash
rtk git status --short 2>/dev/null | head -20   # or: ls -la
find . -maxdepth 2 -type f -not -path "*/.git/*" | head -40
```

| What you find | State | Do |
|---|---|---|
| Empty / only `.git` | **greenfield** | Interview -> write `context/` from the answers. Nothing to move. |
| Loose docs, notes, specs at root - no source tree | **unorganized** | **Run the triage below.** This is the case the user meant. |
| Real source tree (`src/`, `packages/`, `apps/`) | **brownfield** | Do NOT move source. Derive `context/` *from* the code; ask about domains. |
| `context/` already exists and is populated | **ready** | Read it. Skip triage. Confirm it's current, don't assume. |

## Step 1 - triage an unorganized folder

**Ask before moving anything.** Moving a user's files is destructive-ish and surprising; a wrong
guess about what a file *is* is worse than a question. Propose the mapping, get one confirmation,
then move in **one shell call** (`git mv` if tracked - preserve history).

Where things go:

| Source material | Destination | Why |
|---|---|---|
| Vision, product, roadmap, strategy, scope | `context/.initial/` | **Reference-only, never edited.** The original intent, frozen. |
| Architecture / design / system docs | `architecture/` | One decision doc per layer; `docRouting` points at these by filename |
| Requirements, specs, constraints, "how it should work" | `context/architecture/` | Feeds the per-terminal context packs |
| Research, notes, scratch, exploration | `context/.initial/research/` | Kept, but out of the way |
| Existing code | **leave in place** | Brownfield: the layout *is* the domain map |
| Secrets, keys, `.env` | **do not move - flag them** | Never into a tracked dir. `.env.local`, gitignored. Tell the user. |
| Unclear | **ask** | Never guess a destination |

```bash
mkdir -p context/.initial/research context/architecture architecture
git mv VISION.md context/.initial/ 2>/dev/null || mv VISION.md context/.initial/
```

## Step 2 - the context pack is a prerequisite, not an output

After triage, `context/` must answer these before you write the config. If it can't, **ask**  - 
do not infer from prose:

| Question | Feeds | Inferable? |
|---|---|---|
| What is being built, in one paragraph? | `project`, SoT seed | [OK] from `goal:` |
| What are the domains, and who owns which paths? | `terminals{}` | [NO] **ASK** |
| What must be true before what? | `gates{}` | [NO] **ASK** |
| What is sensitive (money/identity/admin/keys)? | `sensitivePaths` | [!] propose, confirm |
| Which doc answers which topic? | `docRouting` | [OK] from files that exist |
| What are the quality/scale requirements? | review depth, arch checklist | [!] propose, confirm |

Then write:

```
context/
+-- README.md              # rule: load ONLY your terminal's file
+-- .initial/              # frozen source material - reference-only, NEVER edited
+-- agents/tN-context.md   # per terminal: you own / current phase / gates / sensitive paths
|                          # sets HARNESS_TERMINAL + CLAUDE_AGENT_NAME
|                          # ENV VAR NAMES ONLY - never a credential value
+-- architecture/          # system-overview.md, module-map.md, security-rules.md
\-- skills/index.md        # GENERATED by sync - do not hand-write
```

## Step 3 - only now scaffold

```bash
bash scripts/scaffold.sh <root> <skill-dir> t1 t2 ...
```

## Why this order is not negotiable

Scaffolding first, then asking, produces a `.bld/` and a `terminals{}` shaped by *your guess*
about a folder you hadn't read. The phase files bake in the wrong ownership, `gate_guard.py`
starts blocking legitimate work on boundaries nobody agreed to, and the fix is a rewrite of the
files the harness treats as the source of truth. **Read the folder, agree the map, then build.**
